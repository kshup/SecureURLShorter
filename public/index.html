URL Shortener Backend Status

The API is running correctly.

Security Measures Implemented:

  • Rate Limiting: Protects against brute-force attacks on login and URL shortening endpoints.
  • Helmet: Sets various HTTP headers to secure the app (e.g., X-Frame-Options, X-Content-Type-Options).
  • CORS (Cross-Origin Resource Sharing): Configured to control which domains can access the API (currently open, should be restricted in production).
  • Mongo Sanitize: Prevents NoSQL query injection attacks by removing malicious characters ($ and .) from request inputs.
  • Google Safe Browsing: Checks submitted URLs against Google's list of unsafe sites before shortening.
  • Input Validation: Validates URL format and IP address format.
  • Authentication (JWT): Securely manages user sessions using JSON Web Tokens.
  • Password Hashing (bcrypt): Stores user passwords securely.
